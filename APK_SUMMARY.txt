APP-DA APK SUMMARY
==================

App name: App-da
Package: com.appda.mobile
Platform: Android APK built with Expo/EAS
Purpose: Disaster awareness, school emergency alerts, mock drill management, student check-in, learning modules, quizzes, emergency resource lookup, and school chat.


1. APK OVERVIEW
---------------

App-da is a React Native Expo mobile app for schools and students.

Main user roles:
- Admin: creates a school, starts mock drills, monitors live drill check-ins, sends emergency alerts, manages profile, views QR check-in code, and deletes alerts.
- Student: joins school by code, receives drill and alert notifications, checks in during drills, studies disaster modules, takes quizzes, uses preparedness checklists, views resources, and joins school chat.
- Guest: can browse learning modules and general app content without school features.

Important APK behavior:
- Remote push notifications work in EAS APK builds, not Expo Go.
- Drill and alert push notifications are sent by Firebase Cloud Functions through Expo Push API.
- Android push requires EAS FCM V1 credentials for package com.appda.mobile.
- Resource map in native APK uses OpenStreetMap image tiles instead of react-native-maps to avoid APK map crashes.

3. IMPORTANT DATA FLOW
----------------------

Login/Register:
- User signs in through Firebase Auth.
- Profile is loaded from users/{uid}.
- APK registers an Expo push token and stores it on the user document.

Drill:
- Admin creates a drills document.
- Cloud Function onDrillCreated finds student tokens for the school.
- Expo Push API sends drill notification.
- Student taps notification and opens drill/student.

Alert:
- Admin creates an alerts document.
- Cloud Function onAlertCreated finds student tokens for the school.
- Expo Push API sends emergency notification.
- Student Alerts page subscribes to the school alerts query.

Map:
- App resolves resources near selected district/location.
- Native APK renders a 3x3 OpenStreetMap tile preview with marker.
- Full map opens externally in Google Maps.

5. BUILD AND DEPLOY COMMANDS
----------------------------

Install dependencies:
npm install

Run development server:
npm run start

Clear Expo cache:
npm run start:clear

Typecheck:
npm run typecheck

Build Android preview APK:
npm run eas:build:android

Deploy Firestore rules and indexes:
npm run db:deploy

Deploy Cloud Functions:
npm run functions:deploy

Deploy backend rules and functions:
npm run backend:deploy


6. CURRENT IMPORTANT IMPLEMENTATION NOTES
-----------------------------------------

- Native map uses image tiles, not react-native-maps, to avoid standalone APK map crashes.
- Alert delete is available for admins in the Alerts screen.
- Alert send returns the created Firestore id and updates admin UI immediately.
- Remote alert and drill notifications require student APK login so expoPushToken is stored.
- Expo Go is not enough for closed-app remote push testing.
- For UI changes in an APK, rebuild and reinstall the APK.
- For server/EAS credential changes, a rebuild is not always needed.
